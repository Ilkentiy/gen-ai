## 1. Валидатор схемы

В рамках работы был реализован валидатор схемы, встроенный в orchestrator.py. Он проверяет план до его исполнения и предотвращает два типа ошибок: использование выдуманных инструментов и отсутствие calculate в арифметических подвопросах.

**Примеры пойманных ошибок:**

| Тип ошибки | Пример | Ошибка валидатора |
|------------|--------|-------------------|
| Выдуманный инструмент | expected_tools=["get_cumulative_inflation"] | Подвопрос 1: неизвестный инструмент 'get_cumulative_inflation' |
| Арифметика без calculate | question="Во сколько раз USD подорожал?" + expected_tools=["get_fx_rate"] | Подвопрос 3: арифметика требует calculate |

**Эффективность:** На Q4 валидатор поднял прохождение с 1/3 до 3/3.


## 2. Параллельное исполнение

```python
# Q1: параллельно
Measure-Command { python orchestrator.py --quiet "Во сколько раз USD подорожал с 1 января 2022 по сегодня?" }

# Q1: последовательно
Measure-Command { python orchestrator.py --quiet --sequential "Во сколько раз USD подорожал с 1 января 2022 по сегодня?" }

# Q4: параллельно
Measure-Command { python orchestrator.py --quiet "Какой курс USD на 1 января 2023 и курс EUR на 1 января 2023?" }

# Q4: последовательно
Measure-Command { python orchestrator.py --quiet --sequential "Какой курс USD на 1 января 2023 и курс EUR на 1 января 2023?" }
```

В orchestrator.py реализовано параллельное исполнение независимых подвопросов через ThreadPoolExecutor.

**Замеры времени:**

| Режим | Q1 (сек) | Q4 (сек) |
|-------|----------|----------|
| Последовательно | 27.92 | 21.70 |
| Параллельно | 31.85 | 17.63 |
| Ускорение | 0.88x | 1.23x |

**Вывод:** На Q1 ускорение не достигнуто из-за накладных расходов. На Q4 ускорение составило 1.23x.


## 3. Замер критики: T=0.0 vs T=0.7

**Метод:** 5 битых кейсов × 10 прогонов при каждой температуре.

**Результаты:**

| Битый кейс | T=0.0 | T=0.7 |
|------------|-------|-------|
| арифметика без calculate | 1/10 | 0/10 |
| выдуманное число | 4/10 | 5/10 |
| несогласованные данные | 5/10 | 7/10 |
| ошибка в ответе | 0/10 | 0/10 |
| неполный план | 0/10 | 0/10 |
| **ИТОГО** | **10/50** | **12/50** |

**Вывод:** Гипотеза «шум лечит зеркальное соглашение» не подтвердилась. При T=0.7 ложных принятий больше (12/50), чем при T=0.0 (10/50).


## 4. Eval: 6 вопросов × 3 конфигурации (N=3)

**Конфигурации:** 1) Одиночный агент, 2) PWC без валидатора, 3) PWC + валидатор.

**Результаты:**

| Вопрос | Single | PWC без валидатора | PWC + валидатор |
|--------|--------|-------------------|-----------------|
| Q1: Во сколько раз USD подорожал с 01.01.2022? | 2/3 | 2/3 | 2/3 |
| Q2: Реальная ключевая ставка по последнему месяцу | 3/3 | 2/3 | 2/3 |
| Q3: Накопленная инфляция 01.2022–03.2026 | 1/3 | 0/3 | 0/3 |
| Q4: Курс USD и EUR на 01.01.2023 | 3/3 | 1/3 | **3/3** |
| Q5: Инфляция в декабре 2023 | 3/3 | 3/3 | 3/3 |
| Q6: Сравнение USD и EUR на сегодня | 3/3 | 0/3 | 0/3 |

**Ключевые наблюдения:**

1. **Валидатор работает:** Q4 — PWC без валидатора 1/3, с валидатором 3/3.
2. **Q3 остаётся проблемой:** все подходы дают 0/3 — граница паттерна.
3. **Q6:** Single — 3/3, PWC — 0/3 — PWC переусложняет задачу.


## 5. Общий вывод

**PWC оправдан, когда:**
- Вопрос требует декомпозиции с зависимостями
- Есть риск галлюцинаций инструментов
- Нужна проверка согласованности (критик)

**Лучше остаться с одиночным агентом, когда:**
- Вопрос простой (1-2 вызова)
- PWC переусложняет задачу
- Время ответа критично

**Итоговый вывод:** PWC — мощный паттерн, но не универсален. Валидатор эффективен (Q4: 1/3 → 3/3). Критик при T=0.7 дал больше ложных принятий (12/50), чем при T=0.0 (10/50). Параллельное исполнение дало ускорение 1.23x только на полностью независимых подвопросах.
