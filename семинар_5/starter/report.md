## 1. Новый инструмент compare_periods

Сигнатура:
def compare_periods(
    metric: str,               # "key_rate" | "fx_USD" | "fx_EUR" | "fx_CNY" | "cpi" | "unemployment"
    period_a: str,             # "YYYY-MM" или "YYYY-MM-DD"
    period_b: str,
) -> dict:

Описание: Сравнивает значение метрики в двух периодах, возвращает delta (разницу) и ratio (отношение).

Примеры запросов:

1. "Во сколько раз вырос курс USD с января 2022 по апрель 2026?"
   compare_periods(metric="fx_USD", period_a="2022-01", period_b="2026-04")
   Результат: a=77.82, b=74.88, ratio=0.9623, delta=-2.94

2. "Как изменилась безработица между январем 2020 и январем 2024?"
   compare_periods(metric="unemployment", period_a="2020-01", period_b="2024-01")
   Результат: error - не удалось получить данные

## 3. Диагностика ошибок

### Ошибка 1: Неправильная интерпретация "во сколько раз" (Q5)

Трасса из trace.jsonl:
{"run_id": "f837f85b-9338-452d-bf45-680360c5f60c", "step": 1, "call": "compare_periods", "args": {"metric": "fx_USD", "period_a": "2022-01", "period_b": "2026-04"}, "obs": {"a": {"value": 77.8174}, "b": {"value": 74.8806}, "ratio": 0.9623, "delta": -2.9368}}
{"run_id": "f837f85b-9338-452d-bf45-680360c5f60c", "step": 2, "final": "Курс доллара не вырос, а снизился... отношение 0,96"}

Причина: Агент понял "во сколько раз вырос" буквально. Так как ratio = 0.9623 < 1, курс фактически упал. Ответ не содержит слово "раз", из-за чего тест провалился.

Что исправить: В описании compare_periods добавить правило: "Если ratio < 1, отвечай: упал в 1/ratio раз"

{"run_id": "3e24e465-84ed-4000-a7db-ed17ac9aaf32", "step": 2, "call": "get_inflation", "args": {"year": 2026, "month": 4}, "obs": {"error": "нет данных ИПЦ на 2026-04"}}
{"run_id": "3e24e465-84ed-4000-a7db-ed17ac9aaf32", "step": 3, "call": "get_inflation", "args": {"year": 2026, "month": 3}, "obs": {"cpi_yoy": 6.8}}

Причина: Запросили май 2026, не нашли данные, затем апрель - тоже нет, взяли март. При этом агент не предупредил пользователя, что использует данные за другой месяц.

Что исправить: В get_inflation при отсутствии данных возвращать ближайшие доступные с флагом "approximate": true и предупреждением в ответе.

## 4. Вывод

Самый хрупкий элемент системы: Качество и полнота данных. CSV-файлы неполные (нет безработицы за 2020 год, нет свежих данных инфляции за апрель-май 2026). Агент не умеет корректно обрабатывать отсутствие данных - впадает в циклы или использует неподходящие данные без предупреждения.

Предложения по улучшению:
1. Добавить fallback-логику во все get_* функции - при отсутствии точных данных искать ближайшие
2. Расширить CSV-файлы историческими данными (безработица с 2020 года, актуальная инфляция)
3. Улучшить системный промпт для вопросов "во сколько раз" - добавить примеры правильных ответов
4. Добавить обязательные предупреждения при использовании приблизительных данных
5. Ограничить количество попыток перебора дат до 3, чтобы избежать бесконечных циклов
